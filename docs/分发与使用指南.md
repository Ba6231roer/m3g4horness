# m3g4h⊿rness 分发与使用指南（企业内网）

> 面向企业内网多业务系统的开发同事：有的用 Claude Code，有的用 opencode。
> 同一个 `m3g4horness/` 工具包服务两类用户，安装时各选一条命令即可。

## 0. 这是什么

`m3g4h⊿rness` 的 **`/mgh-sast`** 是一个**零依赖**的 9 阶段 AI 安全扫描工具（SAST），完整复刻了 vvaharness
流水线（测绘 → 威胁建模 → 拆解 → 深挖 → 预过滤 → 对抗验证 → 去重 → 利用链 → SARIF），
以 **Claude Code / opencode 原生命令**形式运行。支持：

- 整仓扫描
- **增量扫描**：`--diff origin/main`（只扫本次改动 + 直接关联调用链）
- **作用域扫描**：`--path <目录>` / `--package <包名>`（只扫指定范围 + 调用链）
- **批量多仓**：`--repo-file repos.csv`

> ⚠️ 重要：扫描结果是**待人工复核的候选漏洞，不是已确认漏洞**。只扫描你有权扫描的代码。

## 1. 前置条件（每人先自查）

| 条件 | 说明 | 检查命令 |
|---|---|---|
| AI 编程 Agent | Claude Code **或** opencode，已安装并登录企业网关 | `claude --version` / `opencode --version` |
| Python ≥ 3.10 | 跑确定性脚本（SARIF / CVSS / 调用链） | Win：`py --version`；Mac/Linux：`python3 --version` |
| Git | 增量扫描 `--diff` 需要 | `git --version` |

> **无需 `pip install`、无需访问 PyPI / 联网。** 确定性脚本只用 Python 标准库，
> 同事拿到包后不用装任何 Python 第三方包。LLM 凭据/网关由**你自己的 Agent** 管理；
> `m3g4h⊿rness` 不接触、不打印任何密钥。

---

## 2. 获取工具包（内网分发）

内网无外网，由统一渠道获取整个 `m3g4horness/` 目录，三选一：

- **内部 Git**（推荐）：`git clone <内网仓库地址>`
- **内部网盘 / NAS**：下载 `m3g4horness.zip` 解压
- **内部制品库**：拉取打包版本

> 给**分发者**的打包/发布步骤见文末第 10 节。

---

## 3. 安装（按你的 Agent 二选一）

进入**你要扫描的业务代码仓**根目录，把拿到的 `m3g4horness/` 路径准备好，然后选一条命令。
`PATH_TO_MGH` 替换为你实际的 `m3g4horness` 目录路径。

### A. Claude Code 用户 → 装到本仓 `.claude/`

```bash
# macOS / Linux / Git Bash
bash /PATH/TO/m3g4horness/install.sh --claude .
```
```powershell
# Windows PowerShell
.\PATH\TO\m3g4horness\install.ps1 -Platform claude -Target .
```
装完，本仓即可用命令 **`/mgh-sast`**。

### B. opencode 用户 → 装到本仓 `.opencode/`

```bash
# macOS / Linux / Git Bash
bash /PATH/TO/m3g4horness/install.sh --opencode .
```
```powershell
# Windows PowerShell
.\PATH\TO\m3g4horness\install.ps1 -Platform opencode -Target .
```
装完，本仓即可用命令 **`/mgh-sast`**。

> **建议每个业务仓各自安装**（互不污染、版本独立）。想在所有项目通用，把 `.` 换成你的
> 用户主目录即可，但一般不推荐。

---

## 4. 验证安装

在你的 Agent（Claude Code 或 opencode）里直接输入：

```
/mgh-sast
```

应打印参数表与用法（**不消耗 token**）。看到用法说明 = 安装成功。

---

## 5. 首次运行（先不花钱）

先做零花费预览，看规模再决定是否真扫：

```
/mgh-sast --repo . --estimate
```

再做一个控本的小范围真扫（跑到"拆解"阶段就停）：

```
/mgh-sast --repo . --stop-after s3
```

---

## 6. 常用场景

| 目的 | 在 Agent 里输入 |
|---|---|
| 整仓扫描 | `/mgh-sast --repo .` |
| 只扫本次改动 + 调用链 | `/mgh-sast --repo . --diff origin/main` |
| 只扫某目录 + 调用链 | `/mgh-sast --repo . --path src/payment` |
| 只扫某包 + 调用链 | `/mgh-sast --repo . --package com.bank.payment` |
| 批量多仓（每应用一份报告） | `/mgh-sast --repo-file repos.csv --group-by-app` |
| 控本/调试，跑到 s3 停 | `/mgh-sast --repo . --stop-after s3` |

**产物**在 `<目标仓>/security-scan/`：

- `report.md` — 人读报告（发现 + 利用链 + 丢弃发现附录 + 盲区披露）
- `report.sarif` — SARIF 2.1.0（接安全平台 / DefectDojo 等）
- `*_errors.jsonl` — 非致命错误

---

## 7. 重要注意事项（务必读）

1. **只扫你有权的代码**。本工具以较高权限运行，仅对可信、授权仓库使用。
2. **结果是候选，非定论**。需人工复核；同一仓两次扫描可能不同。
3. **Spring 框架盲区**（对银行 Spring 系统尤其注意）：调用图是文本/AST 级，
   `@RequestMapping` / Feign / AOP 切面 / `@Autowired` 注入 / JPA 等动态分派**可能漏**；
   报告会列出"未解析调用"，需人工补查。
4. **成本**：LLM 阶段花真实 token。先用 `--estimate`，大仓用 `--path` / `--diff` / `--stop-after` 收窄。
5. **凭据安全**：密钥/网关配置在**你自己的 Agent 侧**；本工具不处理也不打印密钥。

## 8. 更新与卸载

- **更新**：从内网渠道拿新版 `m3g4horness/`，在业务仓里**重跑同样的 install 命令**（覆盖旧文件）。
- **卸载**：删除业务仓里对应平台的文件——
  - Claude：`.claude/commands/mgh-sast.md`、`.claude/agents/sast-*.md`、`.claude/skills/sast-*`、`.claude/mgh-core`
  - opencode：`.opencode/command/mgh-sast.md`、`.opencode/agent/sast-*.md`、`.opencode/mgh-core`

## 9. 常见问题（FAQ）

- **Windows 上 `python` 报错 / 弹应用商店**：用 `py` 启动器（Python 官方安装包自带），
  `py --version` 验证。命令脚本已用 `py` 调用。
- **macOS / Linux 上 `py` 不存在**：装好的命令文件里调用脚本用的是 `py`。若没有 `py`，
  把 `.claude/commands/mgh-sast.md`（opencode 则 `.opencode/command/mgh-sast.md`）里的
  `py ` 替换成 `python3 `；或建一个别名 `alias py=python3`。
- **`/mgh-sast` 找不到**：确认你在业务仓根目录、且该仓 `.claude/`（或 `.opencode/`）里已装入对应文件。
- **增量扫描提示"无可扫描变更"**：`--diff <ref>` 的 ref 必须是有效 git 引用（如 `origin/main`、`HEAD~3`），
  且确有**源码**改动（纯文档/`.md` 改动不计入）。
- **报告在哪**：`<目标仓>/security-scan/report.md` 与 `report.sarif`。
- **怎么知道用的是哪版**：`cat m3g4horness/VERSION`（或看 `CHANGELOG.md`）。0.x 版为初始开发期，
  结构完整但端到端实跑仍在完善；1.0 为首个稳定版（待 live 验证收尾）。
- **企业网关 401 / 模型不可达**：这是你 Agent 侧的凭据/网关问题，与本工具无关；找本 Agent 的配置指引。

## 10. 分发者须知（给统一发布本工具的人）

1. **打包**：把整个 `m3g4horness/` 目录打成 `m3g4horness.zip` / `m3g4horness.tar.gz`，或推到内网 Git 仓库。
2. **零依赖自检**（发布前在包根目录跑，应**无输出**）：
   ```bash
   grep -rnE "^[[:space:]]*(import[[:space:]]+vvaharness|from[[:space:]]+vvaharness[[:space:]]+import)" --include=*.py .
   ```
3. **版本标识**：包内已含 `VERSION`（当前 `0.1.1`，0.x 表示初始开发期）与 `CHANGELOG.md`。
   同事可用 `cat m3g4horness/VERSION` 查看所用版本；发新版时更新这两者并在 CHANGELOG 写明变更。
4. **预置凭据指引**：附上企业网关 / 模型凭据的配置说明（属各 Agent，不属本工具）。
5. **更新通知**：发新版时通知同事"重跑 install 覆盖"；重大变更在 `CHANGELOG` 里写明。
6. **提示词来源**：本工具提示词自 vvaharness（Apache-2.0）移植，保留署名；分发即代表合规（见包内 `core/docs/NOTICE`）。
