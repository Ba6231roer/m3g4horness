# /mgh-sast report format

Assemble `security-scan/report.md` from the stage artifacts into EXACTLY this
structure. The report is the primary human-facing artifact.

## Required structure

```markdown
# Security scan — <repo-name>

> **Triage candidates, not confirmed vulnerabilities.**
> Findings below are LLM-generated leads requiring human review. Runs are
> non-deterministic. Generated by /mgh-sast (zero-dependency reimplementation
> of the vvaharness pipeline; prompts ported under Apache-2.0).

**Scope:** <full-repo | incremental vs <ref> | path <dir> | package <pkg>>
**Target git SHA:** <sha>   **Model profile:** <profile>   **Run:** <ts>

## Summary
| Severity | Count |
|---|---|
| Critical | n |
| High | n |
| Medium | n |
| Low | n |
| Info | n |

## Findings
### [F-001] <title>  · <severity> · <cwe>
- **CVSS:** <score> <vector>
- **Location:** `source_ref` → `sink_ref`
- **Impact:** <impact>
- **Exploit scenario:** <exploit_scenario>
- **Preconditions:** <…>
- **Recommendation:** <recommendation>
- **Confidence:** <confidence>

(repeat per finding, severity-descending)

## Exploit chains
### CH-1 (<rank>)
<step F-001> → <step F-003>: <narrative>

## Dropped findings (appendix)
| ID | Title | Reason dropped |
|---|---|---|
| F-NN | … | <prefilter/dedup reason> |

## Limitations & blind spots
- Findings are triage candidates; human review required.
- Call graph is textual/AST-level. These framework-routed/DI classes had **no
  textual edge** to the scan seed and were NOT followed automatically — review
  manually: <list scope_manifest.unresolved[]>
  (Spring `@*Mapping`/Feign/AOP/`@Autowired`/JPA, reflection, dynamic dispatch).
```

## Rules
- Severity labels come straight from the SARIF enrichment (CVSS 3.1 band) —
  never invent or override them.
- Every finding must cite real `source_ref`/`sink_ref` file:line locations.
- The dropped appendix must state the reason each item was dropped.
- Always include the limitations section; if `unresolved[]` is empty, state that
  the textual graph found no unresolved framework edges (still note the
  inherent blind spot).
